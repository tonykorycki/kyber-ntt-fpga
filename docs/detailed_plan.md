## 1. Module Definitions

### 1.1 `hls/src/barrett.cpp` — Barrett Modular Multiplier (HLS)

**Function signature:**
```cpp
coef_t barrett_mul(coef_t a, coef_t b);
```

**Types (defined in `hls/src/ntt_engine.h`):**
```cpp
typedef ap_uint<COEF_W>      coef_t;   // 5-bit dev, 12-bit Kyber
typedef ap_uint<2 * COEF_W>  prod_t;   // intermediate product width
```

**Constants (precomputed for each (n, q) pair, stored in `ntt_engine.h`):**
| Constant | Dev (q=17) | Full Kyber (q=3329) | Formula |
|---|---|---|---|
| `BARRETT_K` | 5 | 12 | `q.bit_length()` = `ceil(log2(q))` |
| `BARRETT_M` | 60 | 5039 | `floor(1 << (2*K)) / q` |

**Algorithm:**
```
prod_t ab       = (prod_t)a * (prod_t)b
prod_t q_approx = (ab * BARRETT_M) >> (2 * BARRETT_K)
coef_t result   = ab - q_approx * Q
if (result >= Q): result -= Q   // single correction, fires at most once
return result
```

**HLS pragma:** `#pragma HLS INLINE` — used as a building block; synthesized inline wherever called.

**Timing:** 1 cycle latency after inlining into parent functions.

### 1.2 `hls/src/ntt_engine.cpp` — CT/GS Butterfly Engine (HLS)

**Function signature:**
```cpp
void ntt_engine(coef_t a[N], bool inverse);
```

**Inputs / Outputs:**
| Port | Direction | Width | Description |
|---|---|---|---|
| `a` | in/out | N × COEF_W bits | Coefficient array, modified in place |
| `inverse` | in | 1 bit | false = forward CT NTT, true = inverse GS NTT + INV_N scaling |

**Algorithm (forward, Cooley-Tukey — FIPS 203 Alg 9):**
```
k = 0
for len in N/2, N/4, ..., 2:          // log2(N/2) stages
    for start in 0, 2*len, ...:        // N/(2*len) groups per stage
        zeta = TWIDDLE[k++]
        for j in 0..len-1:
            t              = barrett_mul(zeta, a[start+j+len])
            a[start+j+len] = (a[start+j] - t + Q) % Q
            a[start+j]     = (a[start+j] + t) % Q
```
Inverse (Gentleman-Sande — FIPS 203 Alg 10) traverses the same `TWIDDLE` table in reverse
(`k` counts down from N/2−2 to 0), using positive twiddle values. After all stages, scales
every coefficient by `INV_N = (N/2)^{-1} mod Q`.

**HLS pragmas:**
```cpp
#pragma HLS PIPELINE II=1                              // on inner j-loop
#pragma HLS ARRAY_PARTITION variable=a cyclic factor=2 dim=1
```

**Twiddle source:** `TWIDDLE[N/2]` and `INV_N` from `hls/src/twiddle_rom.h` (generated by
`scripts/gen_twiddle_rom.py`). Only 127 of the 128 slots are accessed at N=256 (1+2+4+…+64
groups across 7 stages); slot 127 is unused padding. The inverse NTT reuses the same table —
no separate inverse twiddle array.

### 1.3 `hls/src/mul_ntt.cpp` — Base-Case Slot Multiply (HLS)

**Function signature:**
```cpp
void mul_ntt(const coef_t A[N], const coef_t B[N], coef_t C[N]);
```

**Background:** After the forward NTT, output pairs `(A[2i], A[2i+1])` represent elements of
`Z_q[x]/(x² − γᵢ)` where `γᵢ = SLOT_ZETA[i] = ζ^{2·brvK(i)+1}`, K = log2(N/2).
Multiplying in this ring requires 5 `barrett_mul` calls per slot.

**Algorithm (N/2 slots, FIPS 203 Alg 11):**
```
for i in 0..N/2-1:
    a0, a1 = A[2i], A[2i+1]
    b0, b1 = B[2i], B[2i+1]
    gamma  = SLOT_ZETA[i]
    C[2i]   = barrett_mul(a0,b0) + barrett_mul(barrett_mul(a1,b1), gamma)  // mod Q
    C[2i+1] = barrett_mul(a0,b1) + barrett_mul(a1,b0)                       // mod Q
```

**HLS pragma:** `#pragma HLS PIPELINE II=1` on the slot loop.

**Twiddle source:** `SLOT_ZETA[N/2]` from `hls/src/twiddle_rom.h`.

### 1.4 `hls/src/ntt_top.cpp` — Top-Level Pipeline + AXI-Lite Owner (HLS)

**Function signature:**
```cpp
void ntt_top(coef_t a[N], coef_t b[N], coef_t c[N]);
```

**AXI-Lite interface (auto-generated by HLS):**
```cpp
#pragma HLS INTERFACE s_axilite port=return bundle=CTRL
#pragma HLS INTERFACE m_axi port=a bundle=MAXI depth=256 offset=slave
#pragma HLS INTERFACE m_axi port=b bundle=MAXI depth=256 offset=slave
#pragma HLS INTERFACE m_axi port=c bundle=MAXI depth=256 offset=slave
#pragma HLS INTERFACE s_axilite port=a bundle=CTRL
#pragma HLS INTERFACE s_axilite port=b bundle=CTRL
#pragma HLS INTERFACE s_axilite port=c bundle=CTRL
```

Arrays a, b, c live in PS DDR. HLS bursts them in/out via the HP0 AXI4 master port. Physical
addresses are passed as AXI-Lite offset registers in the CTRL bundle. PS driver must flush
dcache for a/b before ap_start and invalidate dcache for c after ap_done.

**Generated AXI-Lite register map** (used verbatim by `ps/ntt_driver.c`; never hardcoded elsewhere):

| Offset | Register | Description |
|---|---|---|
| `0x00` | CTRL | bit0=AP_START, bit1=AP_DONE, bit2=AP_IDLE |
| `0x10` | a_1 | lower 32b of physical address of array a |
| `0x14` | a_2 | upper 32b (always 0 on 32-bit PYNQ-Z2) |
| `0x1C` | b_1 | lower 32b of physical address of array b |
| `0x20` | b_2 | upper 32b (always 0) |
| `0x28` | c_1 | lower 32b of physical address of array c |
| `0x2C` | c_2 | upper 32b (always 0) |

**Data packing (CH0_USER_DW=16, AXI data width=32):** two 12-bit coefficients per 32-bit AXI beat,
zero-extended to 16 bits each. N=256 coefficients = 128 beats per array (ARLEN/AWLEN=127), 512 bytes per array.

**Pipeline sequence:**
```
1. ntt_engine(a, false)   // forward NTT on a
2. ntt_engine(b, false)   // forward NTT on b
3. mul_ntt(a, b, c)       // base-case slot multiply (N/2 quadratic slots)
4. ntt_engine(c, true)    // inverse NTT + INV_N scaling
```
Negacyclic structure (reduction mod x^N+1) is baked into the twiddle schedule via odd powers
of ζ — no pre/post twist step needed.

### 1.5 Vivado Block Design (`vivado/ntt_bd.tcl`)

**Purpose:** Recreates the full block design from TCL. No hand-written RTL wrappers — Vivado
auto-generates `ntt_bd_wrapper.v`.

**Components:**
- Zynq PS7 — GP0 master → CTRL; HP0 slave ← HLS MAXI
- `ntt_top_0` — HLS IP (CTRL AXI-Lite + MAXI AXI4 master)
- AXI SmartConnect × 2 — GP0→CTRL, MAXI→HP0

**Address map:**
| Slave | Base | Range |
|---|---|---|
| ntt_top_0 CTRL | 0x4000_0000 | 64K |
| PS HP0 (DDR) | 0x0000_0000 | 512M |

**Clock:** 66.667 MHz from `FCLK_CLK0` via `rst_ps7_0_66M` reset block.

### 1.7 `scripts/gen_twiddle_rom.py` — Twiddle Table Generator

**Inputs (CLI):** `--n N --q Q`

**Outputs:**
- `hls/src/twiddle_rom.h` — C arrays for HLS compile-time use:
  ```cpp

  const coef_t SLOT_ZETA[N/2] = { ... };  // per-slot gammas: ζ^{2·brvK(i)+1}, i=0..N/2-1
  ```

**Derivation:** Uses the same primitive-root computation as `golden/kyber_ntt.py:KyberNTTConfig.from_params()`. Output values must match `golden/kyber_ntt.py` exactly.

## 2. Testbench Definition

### 2.1 Golden Model

**File:** `golden/kyber_ntt.py` — complete Python reference implementation. Source of truth for all hardware verification.

**Tests** (`golden/test_kyber_ntt.py`):
- `test_barrett`: 10 000 random pairs, both dev and Kyber-256 params
- `test_constants`: Barrett K/M, INV_N, zeta order checks
- `test_bit_revK`: all 128 values for N=256
- `test_ct_butterfly` / `test_gs_butterfly`: single butterfly correctness
- `test_twiddle_schedule_count`: exactly 127 twiddles consumed per NTT direction
- `test_twiddle_schedule_values`: spot-checks against FIPS 203 known values
- `test_ntt_zero` / `test_ntt_spikes` / `test_ntt_linearity`: structural NTT properties
- `test_roundtrip`: INTT(NTT(a)) = a (50 random polynomials)
- `test_base_case_mul`: per-slot formula (200 random cases)
- `test_poly_mul`: NTT-mul matches schoolbook negacyclic multiply (50 random pairs)

**Test vectors:** `golden/test_vectors.txt` — triples `(a, b, c=poly_mul(a,b))` generated by
`golden/gen_test_vectors.py`. Format: one polynomial per line, space-separated coefficients.
Used by all HLS and SV testbenches.

### 2.2 HLS C-Simulation Testbenches (`hls/tb/`)

**`tb_barrett.cpp` — Exhaustive Barrett verification**
- For all (a, b) ∈ ℤ_q²: assert `barrett_mul(a, b) == (a * b) % Q`
- 289 pairs for q=17; reports PASS count and any FAIL
- No file I/O needed — all values computed inline

**`tb_ntt_engine.cpp` — NTT engine verification**
- Reads `golden/test_vectors.txt`
- Roundtrip test: `ntt_engine(a, false)` then `ntt_engine(a, true)` recovers original `a`
- Full multiply: NTT(a) → mul_ntt(a,b,c) → INTT(c), compare against schoolbook reference

**`tb_mul_ntt.cpp` — Base-case slot multiply verification**
- For each test vector pair (a, b): verifies per-slot formula against `C[2i]`, `C[2i+1]`
- Reference computed inline using integer arithmetic

**`tb_ntt_top.cpp` — Full pipeline verification**
- Reads `golden/test_vectors.txt`
- For each vector: call `ntt_top(a, b, c_out)`, assert `c_out == c`
- Reports PASS/FAIL per vector; exits with nonzero if any fail (Vitis HLS C-sim convention)

### 2.3 Cocotb Simulation (`sim/test_ntt_top.py`)

**Verification scope:** End-to-end via AXI4 slave memory model: write a/b into simulated DDR →
set pointer registers → AP_START → poll AP_DONE → read c → compare to golden.

**DUT:** HLS-generated Verilog (`vivado/ip_repo/hdl/verilog/ntt_top.v`), simulated with Icarus Verilog.

**Test sequence for each vector:**
1. Write `a[N]`, `b[N]` into dictionary-backed AXI4 slave memory at `0x1000`, `0x2000`
2. Write physical addresses to CTRL offset registers (a_1/b_1/c_1)
3. Assert AP_START; poll AP_DONE (timeout 40k cycles)
4. Read `c[N]` from simulated memory at `0x3000`
5. Compare each coefficient mod Q against `golden/test_vectors.txt`

**Run:** `cd sim && make` (requires cocotb + iverilog; GNU make on PATH)

### 2.4 Performance Measurement

- Cycle counter starts at AP_START assertion, stops at AP_DONE
- Expected: O(N log N) butterfly operations — log2(N/2) stages × N/2 butterflies per stage, plus N/2 base-case slot multiplications
- Baseline comparison: Python schoolbook on ARM Cortex-A9 (measured in `ps/main.cpp`)

## 4. PS Driver (`ps/ntt_driver.c`)

**Purpose:** Standalone C binary for timing-critical hardware access. Jupyter notebook calls
it via `subprocess` for latency measurements and KEM demo.

**Responsibilities:**
1. Open `/dev/mem`, mmap CTRL registers at `0x4000_0000`
2. Allocate physically contiguous CMA buffers for a, b, c (512 bytes each, 16-bit elements)
3. Flush dcache for a/b before writing ap_start (`__builtin___clear_cache` or ARM cache ops)
4. Write physical addresses to CTRL offset registers (a_1/b_1/c_1, upper regs = 0)
5. Assert ap_start, poll ap_done
6. Invalidate dcache for c before reading result
7. Print coefficients or latency to stdout

**Buffer layout:** `uint16_t a[256]`, `uint16_t b[256]`, `uint16_t c[256]` — one coefficient
per 16-bit element (HLS zero-extends 12-bit coef_t to 16 bits; AXI adapter packs 2 per beat).

**Physical address source:** Read from `/proc/self/pagemap` after `mlock()`, or use a reserved
CMA region mapped via `/dev/mem` at a known physical base.

**Build:** `gcc -O2 -o ntt_driver ntt_driver.c`
