## 1. Module Definitions

### 1.1 `hls/barrett.cpp` — Barrett Modular Multiplier (HLS)

**Function signature:**
```cpp
coef_t barrett_mul(coef_t a, coef_t b);
```

**Types (defined in `hls/ntt_engine.h`):**
```cpp
typedef ap_uint<COEF_W>      coef_t;   // 5-bit dev, 12-bit Kyber
typedef ap_uint<2 * COEF_W>  prod_t;   // intermediate product width
```

**Constants (precomputed for each (n, q) pair, stored in `ntt_engine.h`):**
| Constant | Dev (q=17) | Full Kyber (q=3329) | Formula |
|---|---|---|---|
| `BARRETT_K` | 5 | 12 | `ceil(log2(q))` |
| `BARRETT_M` | 60 | `(1<<24)/3329` | `(1 << (2*K)) / q` |

**Algorithm:**
```
prod_t ab       = (prod_t)a * (prod_t)b
prod_t q_approx = (ab * BARRETT_M) >> (2 * BARRETT_K)
coef_t result   = ab - q_approx * Q
if (result >= Q): result -= Q   // single correction, fires at most once
return result
```

**HLS pragma:** `#pragma HLS INLINE` — used as a building block; synthesized inline wherever called.

**Timing:** 1 cycle latency after inlining into parent functions.

### 1.2 `hls/ntt_engine.cpp` — CT/GS Butterfly Engine (HLS)

**Function signature:**
```cpp
void ntt_engine(coef_t coefs[N], const coef_t twiddles[N], bool inverse);
```

**Inputs / Outputs:**
| Port | Direction | Width | Description |
|---|---|---|---|
| `coefs` | in/out | N × COEF_W bits | Coefficient array, modified in place |
| `twiddles` | in | N × COEF_W bits | ω^i (forward) or ω^{-i} (inverse), from `twiddle_rom.h` |
| `inverse` | in | 1 bit | Select GS butterfly + n^{-1} scaling |

**Algorithm (forward, Cooley-Tukey):**
```
bit_reverse_permute(coefs)
for s in 1..log2(N):            // log2(N) stages
    m = 2^s
    for k in 0, m, 2m, ...:    // group start indices
        for j in 0..m/2-1:     // butterfly positions within group
            w    = twiddles[j * N / m]
            t    = barrett_mul(w, coefs[k + j + m/2])
            u    = coefs[k + j]
            coefs[k + j]        = (u + t) % Q
            coefs[k + j + m/2] = (u - t + Q) % Q
```
Inverse uses the same structure with `inv_omega` twiddles and a final `barrett_mul(x, INV_N)` scaling pass.

**HLS pragmas:**
```cpp
#pragma HLS PIPELINE II=1        // on inner j-loop (pipelining across butterfly ops)
#pragma HLS ARRAY_PARTITION variable=coefs complete dim=1   // n=4 only; for n=256 use cyclic
```

**Twiddle source:** constexpr arrays `OMEGA_TABLE[N]`, `INV_OMEGA_TABLE[N]` from `hls/twiddle_rom.h` (generated by `scripts/gen_twiddle_rom.py`).

---

### 1.3 `hls/twist.cpp` — Negacyclic Twist (HLS)

**Function signature:**
```cpp
void twist(coef_t a[N], const coef_t psi_table[N], bool inverse);
```

**Inputs / Outputs:**
| Port | Direction | Description |
|---|---|---|
| `a` | in/out | Coefficient array, modified in place |
| `psi_table` | in | ψ^i (pre-twist) or ψ^{-i} (post-twist), from `twiddle_rom.h` |
| `inverse` | in | Selects PSI_TABLE vs INV_PSI_TABLE at call site |

**Algorithm:** `a[i] = barrett_mul(a[i], psi_table[i])` for i in 0..N-1. N independent multiplications.

**HLS pragma:** `#pragma HLS UNROLL` — all N multiplications issued in parallel (N=4 → 4 parallel `barrett_mul` calls).

---

### 1.4 `hls/pointwise.cpp` — Pointwise Multiply (HLS)

**Function signature:**
```cpp
void pointwise_mul(const coef_t A[N], const coef_t B[N], coef_t C[N]);
```

**Algorithm:** `C[i] = barrett_mul(A[i], B[i])` for i in 0..N-1.

**HLS pragma:** `#pragma HLS UNROLL`

---

### 1.5 `hls/ntt_top.cpp` — Top-Level Pipeline + AXI-Lite Owner (HLS)

**Function signature:**
```cpp
void ntt_top(coef_t a[N], coef_t b[N], coef_t c[N]);
```

**AXI-Lite interface (auto-generated by HLS):**
```cpp
#pragma HLS INTERFACE s_axilite port=return bundle=CTRL  // generates START/DONE registers
#pragma HLS INTERFACE bram port=a                        // BRAM port for input polynomial a
#pragma HLS INTERFACE bram port=b                        // BRAM port for input polynomial b
#pragma HLS INTERFACE bram port=c                        // BRAM port for output polynomial c
```

**Generated AXI-Lite register map** (used verbatim by `ps/ntt_driver.cpp`; never hardcoded elsewhere):

| Offset | Register | Bit | Description |
|---|---|---|---|
| `0x00` | CTRL | [0] | AP_START — write 1 to begin |
| `0x00` | CTRL | [1] | AP_DONE — reads 1 when complete |
| `0x00` | CTRL | [2] | AP_IDLE — reads 1 when waiting |

**Pipeline sequence (dataflow):**
```
1. twist(a, PSI_TABLE,     false)   // pre-twist a
2. twist(b, PSI_TABLE,     false)   // pre-twist b
3. ntt_engine(a, OMEGA_TABLE,     false)  // forward NTT on a
4. ntt_engine(b, OMEGA_TABLE,     false)  // forward NTT on b
5. pointwise_mul(a, b, c)               // pointwise product
6. ntt_engine(c, INV_OMEGA_TABLE, true)  // inverse NTT (INTT)
7. twist(c, INV_PSI_TABLE, true)        // post-twist c
```

**HLS pragma:** `#pragma HLS DATAFLOW` — allows stages to overlap in hardware where data dependencies permit.

### 1.6 `rtl/ntt_top_wrapper.sv` — Structural Top-Level (RTL)

**Purpose:** Pure structural wiring. Instantiates the HLS IP block, three dual-port BRAMs, and `bram_mux`. No logic (`always` blocks), no computation.

**Ports:**
| Signal | Direction | Width | Description |
|---|---|---|---|
| `aclk` | in | 1 | System clock |
| `aresetn` | in | 1 | Active-low reset |
| `s_axi_*` | in/out | AXI4-Lite | AXI-Lite slave — to PS via AXI interconnect |
| `bram_porta_*` | in/out | BRAM | PS BRAM port for BRAM_A, B, C (write inputs / read output) |

**Instantiates:**
- `ntt_top_0` — HLS IP block (contains all compute and AXI-Lite slave registers)
- `blk_mem_gen_0/1/2` — dual-port BRAMs (A, B, C): depth=N, width=COEF_W
- `bram_mux_0` — port ownership arbitrator

---

### 1.7 `rtl/bram_mux.sv` — BRAM Port Arbitrator (RTL)

**Purpose:** The only RTL file with combinational logic. 2:1 mux on BRAM port B: routes either the PS AXI-BRAM signals or the HLS IP signals to the physical BRAM, based on whether the HLS IP is idle.

**Ports:**
| Signal | Direction | Description |
|---|---|---|
| `ps_own` | in | 1 = PS owns BRAM (HLS idle/done), 0 = HLS owns |
| `ps_addr`, `ps_din`, `ps_we` | in | PS BRAM write signals |
| `hls_addr`, `hls_din`, `hls_we` | in | HLS IP BRAM read/write signals |
| `bram_addr`, `bram_din`, `bram_we` | out | Physical BRAM port B signals |

**Control:** `ps_own` is driven by the `ap_idle` output of the HLS IP block.

---

### 1.8 `scripts/gen_twiddle_rom.py` — Twiddle Table Generator

**Inputs (CLI):** `--n N --q Q --outdir DIR`

**Outputs:**
- `hls/twiddle_rom.h` — C++ constexpr arrays for HLS compile-time use:
  ```cpp
  constexpr coef_t OMEGA_TABLE[N]     = { ... };
  constexpr coef_t INV_OMEGA_TABLE[N] = { ... };
  constexpr coef_t PSI_TABLE[N]       = { ... };
  constexpr coef_t INV_PSI_TABLE[N]   = { ... };
  constexpr coef_t INV_N              = ...;
  ```
- `vivado/twiddle_a.coe`, `vivado/twiddle_b.coe` — Vivado BRAM initialization files (for n=256 hardware, where constexpr arrays would be too large for LUT ROM).

**Derivation:** Uses the same primitive-root computation as `golden/ntt.py:NTTConfig.from_params()`. Output values must match `golden/ntt.py` exactly.

## 2. Testbench Definition

### 2.1 Golden Model

**File:** `golden/ntt.py` — complete Python reference implementation. Source of truth for all hardware verification.

**Tests already passing** (`golden/test_ntt.py`):
- `test_barrett`: exhaustive 289 pairs in ℤ₁₇²
- `test_pre_post_twist`: roundtrip = identity (50 random polynomials)
- `test_ntt_forward`: CT butterfly matches naive O(n²) definition (50 random)
- `test_ntt_roundtrip`: INTT(NTT(a)) = a (50 random)
- `test_ntt_mul`: NTT-mul matches schoolbook negacyclic multiply (50 random pairs)

**Test vectors:** `golden/test_vectors.txt` — 16 triples `(a, b, c=ntt_mul(a,b))` at n=4, q=17.
Format: one polynomial per line, space-separated coefficients. Used by all HLS and SV testbenches.

### 2.2 HLS C-Simulation Testbenches (`hls/tb/`)

**`tb_barrett.cpp` — Exhaustive Barrett verification**
- For all (a, b) ∈ ℤ_q²: assert `barrett_mul(a, b) == (a * b) % Q`
- 289 pairs for q=17; reports PASS count and any FAIL
- No file I/O needed — all values computed inline

**`tb_ntt_engine.cpp` — NTT engine verification**
- Reads `golden/test_vectors.txt`
- For each vector: extract `a`, call `ntt_engine(a_copy, OMEGA_TABLE, false)`, compare to expected NTT(a) from a precomputed reference
- Roundtrip test: `ntt_engine(ntt_engine(a, ω, false), ω_inv, true) == a`

**`tb_ntt_top.cpp` — Full pipeline verification**
- Reads `golden/test_vectors.txt`
- For each of 16 vectors: call `ntt_top(a_copy, b_copy, result)`, assert `result == c`
- Reports PASS/FAIL per vector; exits with nonzero if any fail (Vitis HLS C-sim convention)

### 2.3 SystemVerilog Simulation (`sim/tb_ntt_top.sv`)

**Verification scope:** End-to-end: PS writes a/b to BRAM → START → DONE → read c from BRAM.

**Test sequence for each vector:**
1. Write `a[0..N-1]` to BRAM_A via AXI-Lite BRAM port
2. Write `b[0..N-1]` to BRAM_B via AXI-Lite BRAM port
3. Write `0x1` to AXI-Lite CTRL register (AP_START)
4. Poll CTRL register until AP_DONE bit is set (timeout after 1000 cycles)
5. Read `c[0..N-1]` from BRAM_C
6. Compare to expected `c` from `golden/test_vectors.txt`

**Test vectors:** loaded from `golden/test_vectors.txt` via `$fopen`/`$fscanf`.

**Pass/fail:** `$display("PASS")` / `$fatal` per vector. Simulation exits with error if any vector fails.

### 2.4 Performance Measurement

- Cycle counter starts at AP_START assertion, stops at AP_DONE
- Expected: O(N log N) butterfly operations = 8 butterflies for n=4 + N independent twist/pointwise muls
- Baseline comparison: Python schoolbook on ARM Cortex-A9 (measured in `ps/main.cpp`)
