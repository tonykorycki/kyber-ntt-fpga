## 1. Module Definitions

### 1.1 `hls/src/barrett.cpp` â€” Barrett Modular Multiplier (HLS)

**Function signature:**
```cpp
coef_t barrett_mul(coef_t a, coef_t b);
```

**Types (defined in `hls/src/ntt_engine.h`):**
```cpp
typedef ap_uint<COEF_W>      coef_t;   // 5-bit dev, 12-bit Kyber
typedef ap_uint<2 * COEF_W>  prod_t;   // intermediate product width
```

**Constants (precomputed for each (n, q) pair, stored in `ntt_engine.h`):**
| Constant | Dev (q=17) | Full Kyber (q=3329) | Formula |
|---|---|---|---|
| `BARRETT_K` | 5 | 12 | `q.bit_length()` = `ceil(log2(q))` |
| `BARRETT_M` | 60 | 5039 | `floor(1 << (2*K)) / q` |

**Algorithm:**
```
prod_t ab       = (prod_t)a * (prod_t)b
prod_t q_approx = (ab * BARRETT_M) >> (2 * BARRETT_K)
coef_t result   = ab - q_approx * Q
if (result >= Q): result -= Q   // single correction, fires at most once
return result
```

**HLS pragma:** `#pragma HLS INLINE` â€” used as a building block; synthesized inline wherever called.

**Timing:** 1 cycle latency after inlining into parent functions.

### 1.2 `hls/src/ntt_engine.cpp` â€” CT/GS Butterfly Engine (HLS)

**Function signature:**
```cpp
void ntt_engine(coef_t a[N], bool inverse);
```

**Inputs / Outputs:**
| Port | Direction | Width | Description |
|---|---|---|---|
| `a` | in/out | N Ã— COEF_W bits | Coefficient array, modified in place |
| `inverse` | in | 1 bit | false = forward CT NTT, true = inverse GS NTT + INV_N scaling |

**Algorithm (forward, Cooley-Tukey â€” FIPS 203 Alg 9):**
```
k = 0
for len in N/2, N/4, ..., 2:          // log2(N/2) stages
    for start in 0, 2*len, ...:        // N/(2*len) groups per stage
        zeta = TWIDDLE[k++]
        for j in 0..len-1:
            t              = barrett_mul(zeta, a[start+j+len])
            a[start+j+len] = (a[start+j] - t + Q) % Q
            a[start+j]     = (a[start+j] + t) % Q
```
Inverse (Gentleman-Sande â€” FIPS 203 Alg 10) traverses the same `TWIDDLE` table in reverse
(`k` counts down from N/2âˆ’2 to 0), using positive twiddle values. After all stages, scales
every coefficient by `INV_N = (N/2)^{-1} mod Q`.

**HLS pragmas:**
```cpp
#pragma HLS PIPELINE II=1                              // on inner j-loop
#pragma HLS ARRAY_PARTITION variable=a cyclic factor=2 dim=1
```

**Twiddle source:** `TWIDDLE[N/2]` and `INV_N` from `hls/src/twiddle_rom.h` (generated by
`scripts/gen_twiddle_rom.py`). Only 127 of the 128 slots are accessed at N=256 (1+2+4+â€¦+64
groups across 7 stages); slot 127 is unused padding. The inverse NTT reuses the same table â€”
no separate inverse twiddle array.

### 1.3 `hls/src/mul_ntt.cpp` â€” Base-Case Slot Multiply (HLS)

**Function signature:**
```cpp
void mul_ntt(const coef_t A[N], const coef_t B[N], coef_t C[N]);
```

**Background:** After the forward NTT, output pairs `(A[2i], A[2i+1])` represent elements of
`Z_q[x]/(xÂ² âˆ’ Î³áµ¢)` where `Î³áµ¢ = SLOT_ZETA[i] = Î¶^{2Â·brvK(i)+1}`, K = log2(N/2).
Multiplying in this ring requires 5 `barrett_mul` calls per slot.

**Algorithm (N/2 slots, FIPS 203 Alg 11):**
```
for i in 0..N/2-1:
    a0, a1 = A[2i], A[2i+1]
    b0, b1 = B[2i], B[2i+1]
    gamma  = SLOT_ZETA[i]
    C[2i]   = barrett_mul(a0,b0) + barrett_mul(barrett_mul(a1,b1), gamma)  // mod Q
    C[2i+1] = barrett_mul(a0,b1) + barrett_mul(a1,b0)                       // mod Q
```

**HLS pragma:** `#pragma HLS PIPELINE II=1` on the slot loop.

**Twiddle source:** `SLOT_ZETA[N/2]` from `hls/src/twiddle_rom.h`.

### 1.4 `hls/src/ntt_top.cpp` â€” Top-Level Pipeline (HLS)

**Function signature:**
```cpp
void ntt_top(coef_t a[N], coef_t b[N], coef_t c[N]);
```

**Interface pragmas:**
```cpp
#pragma HLS INTERFACE ap_ctrl_hs port=return
#pragma HLS INTERFACE ap_memory  port=a storage_type=ram_1p
#pragma HLS INTERFACE ap_memory  port=b storage_type=ram_1p
#pragma HLS INTERFACE ap_memory  port=c storage_type=ram_1p
```

Arrays a, b, c live in on-chip True Dual-Port BRAM. HLS drives Port B of each BRAM via the
`ap_memory` interface; Port A is owned by the PS via AXI BRAM Controllers. Temporal
arbitration: PS accesses BRAM only while `ap_idle` is high; HLS owns Port B during computation.

Control signals (`ap_start`, `ap_done`, `ap_idle`) are exposed as hardware pins by `ap_ctrl_hs`
and routed through an AXI GPIO IP, allowing the PS to pulse `ap_start` and poll `ap_idle`
via memory-mapped GPIO reads.

**Pipeline sequence:**
```
1. ntt_engine(a, false)   // forward NTT on a
2. ntt_engine(b, false)   // forward NTT on b  (HLS auto-parallelises these two)
3. mul_ntt(a, b, c)       // base-case slot multiply (N/2 quadratic slots)
4. ntt_engine(c, true)    // inverse NTT + INV_N scaling
```
HLS creates two `ntt_engine` instances running NTT(a) and NTT(b) in parallel, so total latency
is `max(NTT_a, NTT_b) + mul_ntt + INTT` â€” not three sequential calls.

Negacyclic structure is baked into the twiddle schedule via odd powers of Î¶ â€” no pre/post
twist step needed.

### 1.5 Vivado Block Design (`vivado/ntt_bd.tcl`)

**Purpose:** Recreates the full block design from TCL. No hand-written RTL â€” Vivado
auto-generates `ntt_bd_wrapper.v`.

**Components:**
- Zynq PS7 â€” GP0 master â†’ AXI SmartConnect â†’ 4 AXI slaves
- `ntt_top_0` â€” HLS IP (`ap_ctrl_hs` control + `ap_memory` BRAM Port B)
- 3Ã— Block Memory Generator â€” True Dual-Port, 256Ã—32-bit, BRAM Controller mode (BRAM A/B/C)
- 3Ã— AXI BRAM Controller â€” PS Port A access to each BRAM
- 1Ã— AXI GPIO â€” CH1 out: ap_start; CH2 in: ap_done (bit 0), ap_idle (bit 1)
- 4Ã— xlconcat â€” 3Ã— replicate `x_we0` â†’ `web[3:0]` byte-write enables; 1Ã— concat ap_done+ap_idle â†’ GPIO CH2

| Peripheral | Base | Range |
|---|---|---|
| AXI BRAM Controller A (polynomial a) | 0x40000000 | 8K |

## 4. PS Driver (`ps/ntt_driver.c`)

**Purpose:** Standalone C binary for hardware access and latency measurement. Jupyter notebook
(`ps/kyber_demo.ipynb`) calls it via `subprocess` with the `-r` flag for low-overhead batch
operation.

**Responsibilities:**
1. Open `/dev/mem`, mmap BRAM A/B/C and GPIO at their physical addresses
2. Write `a[N]`, `b[N]` to BRAM A, BRAM B as 32-bit words (one coefficient per word)
3. Set GPIO CH1 tri-state to output; CH2 tri-state to input
4. Pulse `ap_start` via GPIO CH1 bit 0 (assert then deassert in one write cycle)
5. Poll GPIO CH2 bit 1 (`ap_idle`): wait for deassert (HLS started), then wait for re-assert (done)
6. Read `c[N]` from BRAM C
7. Print coefficients or latency to stdout; report timing via `clock_gettime(CLOCK_MONOTONIC)`

**BRAM layout:** one coefficient per 32-bit word (lower 12 bits used), depth 256.
Access pattern: `bram_a[i] = (uint32_t)a[i]`. Stack-allocated coefficient arrays; no DMA buffers.

**GPIO register map (base 0x40010000):**
| Offset | Register | Use |
|---|---|---|
| `0x00` | CH1 data | write bit 0 to pulse ap_start |
| `0x04` | CH1 tri | set to 0x0 (all outputs) |
| `0x08` | CH2 data | read bit 1 for ap_idle, bit 0 for ap_done |
| `0x0C` | CH2 tri | set to 0x3 (all inputs) |

**Build:** `gcc -O2 -o ntt_driver ntt_driver.c` (no `-lcma`, no `-lpthread`)
